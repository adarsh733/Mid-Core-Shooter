## What It Does

Load any match from 5 days of real LILA BLACK telemetry. See every player path drawn on the actual minimap. Watch the match replay at your own pace. Toggle heatmaps to reveal kill clusters, death zones, and ignored map areas — all in a single browser tab, designed for level designers who want map intuition, not raw tables.

## Features

### Core Requirements

| Feature | What was built |
|---|---|
| Parquet data pipeline | `preprocess.py` reads all `.nakama-0` files, decodes bytes, transforms coordinates, outputs structured JSON |
| Player paths on minimap | Per-map world → UV → pixel transform using scale/origin from README. Y-axis flipped to match image origin |
| Humans vs bots | Visually distinct: cyan paths + larger line weight for humans, rose/pink for bots. Labeled in match list with H/B badges |
| Event markers | Kill `✕` (red), Death `▼` (orange), Loot `◆` (green), Storm `⚡` (purple) — each a distinct shape and colour |
| Filter by map / date / match | Map tabs + date dropdown + scrollable match list with numbered display names |
| Timeline playback | Play/Pause, speed 0.1×–5×, scrubber, auto-stop at end |
| Heatmap overlays | Traffic, Kill, and Death heatmaps using native browser blur for smooth gradients. Separate "all-matches" mode |
| Hosted shareable link | Static deployment — zero backend, zero cost |

### Beyond the Spec

| Extra Feature | Why a Level Designer Benefits |
|---|---|
| **Zoom & Pan** (scroll to zoom, drag to pan, double-click to reset) | Large maps have detail that gets lost at full zoom. Designers need to inspect a specific chokepoint without leaving the tool |
| **All-matches heatmap aggregation** | A single match has too few events to reveal patterns. Aggregating across all matches for a date gives a statistically meaningful view of map flow |
| **Match quality filter — "Top Rated"** | Filters out bot-only and single-player matches. Sorts by lobby size so the designer sees the most representative data first |
| **Match search** | 500+ matches across 5 days. Instant text search by match ID or display label |
| **Loop / Replay button** | Replay a specific engagement multiple times to study positioning without re-selecting the match |
| **Looting rate %** | Percentage of players who picked up at least one item. A single number that flags loot placement issues immediately |
| **Avg distance traveled** | Normalized movement distance per player. Surfaces camping, fast-death patterns, and unexplored areas at a glance |
| **Lobby composition tags** | Matches are labeled "Human-heavy", "Bot-heavy", or "Bot-only" so designers know sample quality before clicking |
| **Light / Dark theme toggle** | Tools get used in dark studios and bright meeting rooms |
| **Mobile responsive layout** | Sidebar stacks vertically on small screens — useful for tablet reviews in a playtest session |
| **Animated tooltip on hover** | Hovering any event marker shows event type, player type, truncated ID, and timestamp |
| **Loading progress indicator** | Visual feedback while large maps load — feels like a real product, not a prototype |

## Tech Stack

| Layer | Choice | Why |
|---|---|---|
| Data pipeline | Python + pyarrow + pandas | Reads parquet natively, runs once, outputs clean JSON |
| Frontend | Vanilla HTML + Canvas API | No build step, no npm, deploys anywhere in seconds |
| Hosting | Netlify or GitHub Pages | Free, static, one-click shareable URL |
| Data format | Pre-baked JSON | Zero latency, browser-native, no WASM library needed |

## Setup

### Prerequisites
```bash
pip install pyarrow pandas
```

### Folder structure
```
lila-tool/
├── preprocess.py
├── index.html
├── ARCHITECTURE.md
├── INSIGHTS.md
├── minimaps/
│   ├── AmbroseValley_Minimap.png
│   ├── GrandRift_Minimap.png
│   └── Lockdown_Minimap.jpg
└── data/        ← generated by preprocess.py
```

### Step 1 — Run the preprocessor
```bash
python preprocess.py --data /path/to/player_data --out ./data
```
Takes ~30–60 seconds for the full dataset. Outputs one JSON per map.

### Step 2 — Serve locally
```bash
python -m http.server 8080
```
Open `http://localhost:8080`

> **Must use a local server** — browsers block `file://` fetch requests by default.

### Environment Variables
None. Everything is static files.

## Deployment

### Netlify (recommended — takes 2 minutes)
1. Run `preprocess.py` locally to populate `data/`
2. Push entire project to GitHub
3. Connect the GitHub repo to Netlify
4. Netlify auto-deploys on every push — shareable URL appears immediately

### GitHub Pages
Enable Pages in repo Settings → Pages → `main` branch → root folder. Done.
